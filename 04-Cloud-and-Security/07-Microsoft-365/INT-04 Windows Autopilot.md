# INT-04: Windows Autopilot

> [!abstract] Overview
> Windows Autopilot is a collection of technologies used to set up and pre-configure new devices, getting them ready for productive use. As an IT support engineer, knowing Autopilot is crucial because it eliminates the need for traditional OS imaging, reducing IT workload and providing a seamless "unboxing" experience for end-users. *Autopilot naye laptops ko bina manual setup ke directly company policies aur apps ke saath ready karta hai.*

---
## 🧠 Concept Overview

- **What it is** — Windows Autopilot is a cloud-based deployment technology in Microsoft Intune that provisions Windows devices automatically over the internet.
- **Why it matters** — Traditional imaging (like SCCM/MDT) requires maintaining gold images and drivers. Autopilot uses the OEM-provided Windows installation and transforms it into a business-ready state. *Image banane ka jhanjhat khatam, sidha cloud se policy aur apps aayenge.*
- **Where you see this** — When a new employee joins the company and receives a brand new laptop shipped directly from the manufacturer to their home.

**L1 / L2 / L3 Split:**

| 👨‍💻 Level | 📋 Responsibility |
|---------|-----------------|
| **L1** | Basic task — Check Autopilot enrollment status, guide users through the OOBE (Out-of-Box Experience) screen. |
| **L2** | Configure, fix, escalate kab karta hai — Extract hardware hashes, import devices into Intune, assign Autopilot profiles, troubleshoot profile assignment issues. |
| **L3** | Architecture, design, enterprise-level — Design Autopilot deployment profiles, configure ESP (Enrollment Status Page), integrate with hybrid Azure AD join, manage OEM vendor relationships for automatic hash uploads. |

> [!tip] Seedha Simple Mein
> *Jaise naya phone lene par Google/Apple ID daalte hi sab apps aur settings wapas aa jaati hain, waise hi laptop ke liye Autopilot kaam karta hai. Laptop on karo, email id daalo, aur baaki sab khud setup ho jayega.*

## 🔬 Technical Deep Dive

### 1. The Hardware Hash (Device ID)

> [!info] Key Concept
> The **Hardware Hash** is a unique identifier generated by a Windows device, used to register it with the tenant's Windows Autopilot service. It contains details like the serial number, manufacturer, and hardware specs.

Every device must be known to the tenant before it can use Autopilot. This is done by importing the Hardware Hash. *Device ki kundli hoti hai Hardware Hash, jo Intune ko batati hai ki yeh hamara laptop hai.*

> [!danger] Common Mistake
> Trying to reuse a hardware hash from a motherboard that has been replaced. If the motherboard changes, the hardware hash changes, and the old hash must be deregistered. *Motherboard change matlab naya hardware hash, purana kaam nahi karega.*

### 2. Autopilot Profiles

Profiles determine how the Out-of-Box Experience (OOBE) behaves.
- **User-Driven:** The user enters their credentials.
- **Pre-provisioned (White Glove):** IT or OEM pre-loads apps before shipping to the user.
- **Self-Deploying:** No user credentials needed (useful for kiosks or digital signage).

### 3. Enrollment Status Page (ESP)

ESP tracks the installation of device-targeted and user-targeted applications, policies, and certificates before the user can access the desktop. *ESP woh loading screen hai jo user ko batati hai ki background mein kya kya install ho raha hai.*

## 🛠️ Step-by-Step Lab

> [!warning] Pre-requisites
> - A Windows 10/11 physical or virtual machine.
> - Intune Administrator or Global Admin rights.
> - A Microsoft 365 E3/E5 or Intune Plan 1 license.

### Step 1: Extract the Hardware Hash

Open PowerShell as Administrator on the target device.

```powershell
# Install the Get-WindowsAutopilotInfo script
Install-Script -Name Get-WindowsAutopilotInfo -Force

# Set Execution Policy if needed
Set-ExecutionPolicy -Scope Process -ExecutionPolicy RemoteSigned

# Export the hash to a CSV file
Get-WindowsAutopilotInfo -OutputFile C:\HWHash.csv
```

> [!success] Expected Output
> ```
> Script downloaded and installed.
> C:\HWHash.csv file generated successfully with Device Serial Number and Hash.
> ```

### Step 2: Import the Hash into Intune

1. Go to **Intune admin center** > **Devices** > **Windows** > **Windows enrollment** > **Devices**.
2. Click **Import** and select the `C:\HWHash.csv` file.
3. Wait for the sync to complete. *Hash upload hone ke baad sync karna zaroori hai.*

### Step 3: Create and Assign Autopilot Profile

1. Go to **Devices** > **Windows** > **Windows enrollment** > **Deployment Profiles**.
2. Click **Create profile** > **Windows PC**.
3. Set **Deployment mode** to `User-Driven`.
4. Hide EULA, Privacy settings, and change account options to `Standard User` (not Administrator).
5. Assign to a device group containing your imported device.

### Step 4: Run the OOBE

1. Reset the target device.
2. Connect to the Internet.
3. The "Welcome to [Company Name]" branding should appear instead of the generic Microsoft login.
4. User logs in with their M365 credentials.

## 🚑 Troubleshooting Guide

| ⚠️ Problem | 🔍 Wajah (Cause) | 🛠️ Fix |
|-----------|----------------|-------|
| **Device not showing Company Branding during OOBE** | Device internet se connected nahi hai ya profile assign nahi hui hai. | Shift+F10 press karke cmd open karo, internet check karo. Intune portal mein sync run karo. |
| **"Something went wrong" error code 80180014** | Device limit reached for the user or device already enrolled. | User ki device limit badhao ya purana stale record Intune/Azure AD se delete karo. |
| **Hardware hash import failed** | CSV file format galat hai ya column headers missing hain. | `Get-WindowsAutopilotInfo` script use karke freshly generate karo, manual edit mat karo. |
| **ESP taking too long and failing** | Koi win32 app install hone mein fail ho rahi hai (timeout error). | ESP mein se heavy apps nikal do aur unhe background mein deploy hone do (after login). |

## 🎫 Real-World Ticket Scenarios

### 🎫 Scenario 1: New Hire Cannot Pass OOBE Screen

> [!example] Ticket
> "Hi IT, I received my new laptop today. When I turn it on and enter my email, it says my device is not recognized and asks for a personal Microsoft account."

**L1 Response:** Confirm with the user if they see the company logo. Ask them to connect to Wi-Fi.
**Escalation Trigger:** The device is connected to Wi-Fi but still showing the consumer setup screen.
**L2 Resolution:** Check the hardware vendor's purchase order. The OEM forgot to upload the hardware hash to the tenant. Ask the user to press Shift+F10, walk them through running the Autopilot info script to extract the hash, or contact the vendor to push the hash to Intune. *Agar hash cloud pe nahi hai, toh laptop ko pata hi nahi ki woh company ka hai.*

### 🎫 Scenario 2: Motherboard Replacement Caused Autopilot Failure

> [!example] Ticket
> "The Dell technician replaced the motherboard on my laptop yesterday. Now it's stuck on a setup screen and won't let me log in with my company email."

**L1 Response:** Verify the device serial number. Ask for the Dell service tag.
**Escalation Trigger:** Motherboard was changed. The hardware hash is now entirely different.
**L2 Resolution:** Delete the old Autopilot device record from Intune. Delete the associated Entra ID (Azure AD) device object. Re-register the new motherboard's hardware hash manually. *Naya dil (motherboard), matlab nayi pehchaan. Purana record delete karna padega.*

### 🎫 Scenario 3: Apps Not Installing During ESP

> [!example] Ticket
> "I am setting up my laptop and the 'Setting up your device for work' screen has been spinning for 45 minutes on 'Installing applications (2 of 5)'."

**L1 Response:** Tell the user to wait up to 60 minutes as some apps take time depending on bandwidth.
**Escalation Trigger:** After 60 minutes, the screen times out and shows a red error page.
**L2 Resolution:** Review the Intune device logs. Check which app is blocking the ESP. Often, large apps like AutoCAD or poorly packaged Win32 apps cause timeouts. Recommend removing that specific app from the ESP required list, allowing it to install silently in the background after the user reaches the desktop.

## 🎤 Interview Questions

> [!question] Q1: What is the primary difference between Windows Autopilot and SCCM Task Sequences?
> **Answer:** SCCM Task Sequences completely wipe the drive and apply a custom OS image (WIM file) and drivers. Autopilot uses the existing OEM Windows OS that comes with the device and simply applies configuration policies and apps to make it enterprise-ready.

==**Exam Tip:** Remember that Autopilot is NOT an OS imaging tool; it is a provisioning tool.==

> [!question] Q2: What happens if a device is registered in Autopilot but the user does not have an Intune license?
> **Answer:** The device will show the company branding and prompt for credentials, but after login, the MDM enrollment will fail because the user is not licensed to receive policies or apps from Intune.

> [!question] Q3: A user has a laptop that was stolen. How does Autopilot help?
> **Answer:** If the laptop is wiped or reset by the thief, the next time it connects to the internet, Autopilot will force it to the company's login screen. Without company credentials, the laptop becomes unusable.

> [!question] Q4: What is "Pre-provisioned deployment" (formerly White Glove) in Autopilot?
> **Answer:** It allows IT admins or OEM partners to pre-load all policies, apps, and settings on the device *before* shipping it to the end-user. The user then only needs to log in, and the device is ready almost instantly.

> [!question] Q5: Can Autopilot join devices to on-premises Active Directory?
> **Answer:** Yes, via "Hybrid Azure AD Join" Autopilot profiles. However, it requires an Intune Connector for Active Directory installed on an on-prem server, and the laptop must have a line of sight to the domain controllers (usually via VPN during OOBE).
