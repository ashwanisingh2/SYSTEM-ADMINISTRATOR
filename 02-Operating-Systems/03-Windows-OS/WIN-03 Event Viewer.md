# WIN-03: Event Viewer

> [!abstract] Overview
> Yeh note Event Viewer ko cover karta hai, jo ek centralized logging utility hai. Yeh Windows operating system aur third-party software ke errors, warnings, aur events record karta hai. Ek support engineer ke liye yeh troubleshoot karne ke liye zaroori hai.

---
## 🧠 Concept Overview

- **What it is** — Windows Event Viewer is a centralized logging utility that records system, security, application, and service events, warnings, and errors generated by the Windows operating system and third-party software.
- **Why it matters** — Troubleshooting system errors, security audits, and application crashes without logs is impossible. A support engineer must know how to filter log databases, locate specific Event IDs, create custom views, and export logs to diagnose problems.
- **Where you see this** — Finding the cause of random restarts, identifying account lockout sources, auditing failed logins, diagnosing application crashes, and monitoring device driver errors.

**L1 / L2 / L3 Split:**

| 👨‍💻 Level | 📋 Responsibility |
|---------|-----------------|
| **L1** | Reviews local Application and System logs, checks basic event descriptions, and filters logs by Event IDs. |
| **L2** | Creates custom XML log views, configures Event Subscriptions (Log forwarding), and analyzes crash dump logs. |
| **L3** | Establishes centralized logging systems (SIEM/Log Analytics), writes custom event collection scripts, and audits corporate security compliance logs. |

> [!tip] Seedha Simple Mein
> *Event Viewer computer ka black box hai. Jo bhi computer mein hota hai (chahe app crash ho ya user login), sab details yahan logs banke record hoti hain.*

## 🔬 Technical Deep Dive

### 1. The Core Log Categories

> [!info] Key Concept
> Windows divides events into primary channels such as Application, Security, and System logs to compartmentalize data.

Windows divides events into several primary channels:
- **Application Log**: Records events logged by local applications and programs (e.g., database connection drops or program crashes).
- **Security Log**: Records security audit events, including logon successes and failures, folder access tracking, and privilege assignments. Managed by security audit policies.
- **System Log**: Records events logged by Windows system components, including driver errors, service startup failures, and hardware alerts.
- **Applications and Services Logs**: Dedicated logs for specific Microsoft components (e.g., PowerShell, Group Policy) and third-party software.

> [!danger] Common Mistake
> Searching for application crashes in the System log instead of the Application log.

### 2. Event Severity Levels
- **Information**: Indicates successful operation of a service or task.
- **Warning**: Indicates a potential problem or non-critical event.
- **Error**: Indicates a significant problem that may cause data loss or service degradation.
- **Critical**: Indicates a severe failure that caused a system crash or sudden shutdown.

### 3. Key Event IDs Reference Table
| Event ID | Log Channel | Severity | Meaning | Diagnostic Focus |
|---|---|---|---|---|
| **`41`** | System | Critical | Kernel-Power (Shutdown without clean power-off) | PSU failure, overheating, user held power button |
| **`1014`** | System | Warning | DNS Name Resolution Timeout | Local network drops, DNS server offline |
| **`1074`** | System | Information | System Shutdown/Restart Initiated | Find which user or process triggered a restart |
| **`4624`** | Security | Success Audit | Successful account logon | Verify user authentication and logon type (RDP/Console) |
| **`4625`** | Security | Failure Audit | Failed account logon | Identify password hacking attempts or expired credentials |
| **`4740`** | Security | Success Audit | User account locked out | Find the source machine locking the AD account |
| **`6008`** | System | Error | Unexpected shutdown | Crash, BSOD, or power failure |
| **`7045`** | System | Information | A new service was installed in the system | Auditing unauthorized software installations |

## 🛠️ Step-by-Step Lab

> [!warning] Pre-requisites
> - A Windows 11 VM or physical machine with local administrator rights.

### Step 1: Create a Custom XML View

```powershell
# Retrieve unexpected shutdown events (Event 6008) from the System log
Get-WinEvent -FilterHashtable @{LogName='System'; Id=6008} -MaxEvents 10 |
    Select-Object TimeCreated, Message
```

> [!success] Expected Output
> ```
> PowerShell returns the matching log entries showing time created and details of the unexpected shutdown.
> ```

## 🚑 Troubleshooting Guide

| ⚠️ Problem | 🔍 Wajah (Cause) | 🛠️ Fix |
|-----------|----------------|-------|
| System restarted randomly in middle of day | Domain administrator manually initiated restart to apply updates without coordination | Find Event ID 1074 in System Log. Re-align admin team with change management policies. |
| Active Directory Account keeps locking out | User changed AD password, but a secondary workstation was left running a script with old cached credentials | Query Event ID 4740 on DC to find 'Caller Computer Name'. Locate that machine and log off the disconnected session. |

## 🎫 Real-World Ticket Scenarios

### 🎫 Scenario 1: Identify Which User or Process Restarted a Production Server

> [!example] Ticket
> "The application server rebooted unexpectedly in the middle of the day, disconnecting all active database sessions."

**L1 Response:** Check the System log for basic restart events or error logs.
**Escalation Trigger:** Unclear cause of shutdown or recurring issue.
**L2 Resolution:** Filtered System Log for Event ID 1074. Found that Domain\Administrator initiated planned restart via shutdown.exe to apply updates. Re-aligned with change management policies.

## 🎤 Interview Questions

> [!question] Q1: What does Event ID 41 mean, and what are its common causes?
> **Answer:** Event ID 41 (Kernel-Power) indicates the system rebooted without performing a clean shutdown sequence. Common causes include a user holding down the physical power button, a power supply failure, or a system thermal shutdown caused by overheating.

==**Exam Tip:** Do not assume Event 41 is just a bad PSU. Always check for corresponding Event ID 1001 (Bugcheck) or Event 6008 to find the actual crash cause.==
