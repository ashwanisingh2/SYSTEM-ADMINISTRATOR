# 🛠️ System Administrator & Desktop Support Engineer Knowledge Vault

Welcome to the **System Administrator & Desktop Support Engineer (L1 to L3) Knowledge Vault**. This is a comprehensive, production-grade documentation repository and study resource structured as an **Obsidian Vault**. It covers everything from basic hardware support (L1) to advanced cloud infrastructure and systems administration (L3).

---

## 📁 Repository Structure

The vault is organized into modular directories matching the standard progression of a systems engineer:

| Directory | Description | Key Topics |
| :--- | :--- | :--- |
| [**`00-MOC/`**](./00-MOC) | Maps of Content & References | Master navigation index, Command Cheat Sheets, troubleshooting trees, and Q&A banks. |
| [**`01-Foundations/`**](./01-Foundations) | IT Infrastructure Basics | Computer Hardware, Virtualization (VMware/VirtualBox), and Core Networking (DHCP, DNS, TCP/IP, Subnetting). |
| [**`02-Operating-Systems/`**](./02-Operating-Systems) | OS Administration | Deep-dive Windows OS management and Red Hat Enterprise Linux (RHEL) administration. |
| [**`03-Identity-and-Core-Services/`**](./03-Identity-and-Core-Services) | Directory & Core Services | Windows Server 2022 administration, Active Directory Domain Services (AD DS), Group Policy, and FSMO roles. |
| [**`04-Cloud-and-Security/`**](./04-Cloud-and-Security) | Cloud & Device Management | Microsoft 365, Microsoft Intune (MDM/MAM), Azure Cloud Administration (AZ-104), and Zero-Trust Security. |
| [**`05-Automation-and-Ticketing/`**](./05-Automation-and-Ticketing) | Efficiency & Operations | PowerShell Scripting, ITSM frameworks, Ticketing workflows (SLA/Incident/Change Management). |
| [**`06-Career-Growth/`**](./06-Career-Growth) | Career Development | Hand-on Lab setups, certification roadmaps (RHCSA, CCNA, AZ-104), and interview preparation templates. |

## 🚀 How to Use this Vault

### 1. 📂 Open in Obsidian (Recommended)
This repository is optimized for **[Obsidian](https://obsidian.md/)** (a local markdown-based knowledge management tool) to enable rich cross-linking and graphical node views:
1. Download and install Obsidian.
2. Choose **"Open folder as vault"**.
3. Select this repository's root directory (`SYSTEM ADMINISTRATOR`).
4. Start with the [**`00-MOC/Master-Index.md`**](./00-MOC/Master-Index.md) file to navigate.

### 2. 📖 Standard Markdown Reading
You can also browse this repository directly on GitHub or using any markdown reader. Links are configured using relative paths for seamless navigation.
